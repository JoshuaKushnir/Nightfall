**Description**
Optimize HeavenEnvironmentController’s per-frame grass blade update loop to remove avoidable costs while preserving identical visual output. This includes removing redundant sqrt work, guarding grid scans behind a dirty-cell check, improving blade creation batch behavior, and switching wind target RNG to a seeded Random instance.

**Acceptance Criteria**
- [ ] `math.sqrt` called at most once per blade per frame (remove the shadowed declaration)
- [ ] `updateGrid` has a dirty flag: skips the scan if the player's cell coord hasn't changed since last frame (same pattern as `GrassService._lastGridPos`)
- [ ] `createCell` batch-pre-allocates blades via a helper so the pool-miss clone path is a single `Clone()`, not re-entered per blade
- [ ] `pickNewWindTarget` uses a seeded `Random` object instead of `math.random()`
- [ ] `GrassService` module is deleted (it is never required anywhere)
- [ ] No visual change observable in Studio playtest

**Additional Context**
Scope is pure optimization (no balance/value tuning).  
Target: keep hot loop under 0.5 ms/frame at ~2,000 active blades.
